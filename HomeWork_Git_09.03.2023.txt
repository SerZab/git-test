1. Как скопировать на свой компьютер репозиторий из github.com, что должно быть сделано предварительно?

Для того, чтобы скопировать репозиторий с GitHub.com на свой компьютер, необходимо выполнить следующие шаги:
Зайти в свой аккаунт на https://github.com, или зарегистрироваться.
Создать свой собственный репозиторий на GitHub, например "git-test".
Установить Git на свой компьютер, если он еще не установлен.
Зайти в terminal, перейти в нужную директорию (cd /путь), создать папку (mkdir <name folder>) и затем выполнить команду:
    git clone <link>
Для этого нужно скопировать URL/SSH репозитория, который находится на странице репозитория на GitHub.
Например:

    git clone git@github.com:SerZab/git-test.git
    git clone https://github.com/SerZab/git-test.git

2. Как создать git репозиторий из локального каталога на своем компьютере. Как его отправить на github.com?

Чтобы создать Git репозиторий из локального каталога на компе и отправить его на GitHub.com, нужно:
Инициализируйте Git репозиторий в вашем локальном каталоге, запустив команду в терминале:

    git init

Например, если ваш локальный каталог находится в папке с именем git-test, то откройте терминал и перейдите в папку git-test,
а затем выполните команду git init.
После выполнения проекта, используя команду git add.
Например, если вы хотите добавить все файлы в папке git-test, то запустите команду

    git add .

Закоммитьте изменения, используя команду git commit в терминале или командной строке.
Например, если вы хотите закоммитить все добавленные файлы, то запустите команду:

    git commit -m "initial commit"

Создайте репозиторий на GitHub.com, если вы еще не сделали этого.
Зайдите на GitHub.com, нажмите на кнопку "New" и заполните необходимые поля.

Добавьте удаленный репозиторий GitHub в ваш локальный репозиторий,
используя команду git remote add в терминале или командной строке.

Например, если ваш репозиторий на GitHub называется git-test, то запустите команду

    git remote add origin https://github.com/<your-username>/git-test.git.

Отправьте свои коммиты на GitHub, используя команду git push в терминале или командной строке.
Например, если вы хотите отправить изменения из локального репозитория в ветку main на удаленный репозиторий, то:

    git push -u origin main // первый раз
    git push               // следующие разы

3. Как получить на локальный компьютер изменения с github.com

Получить изменения с GitHub на локальный компьютер.
Для этого нужно выполнить команду git pull origin <название ветки> в командной строке. Например:

    git pull origin main

4. Какую информацию нам дают команда :
   git status
   git log
   git remote -v

- git status сообщает о текущем состоянии рабочего каталога и индекса в вашем локальном репозитории.
Показывает, какие файлы были изменены, добавлены или удалены, но еще не были проиндексированы или зафиксированы.
Также показывает о текущей ветке и обновлениях, которые необходимо сделать.

- git log отображает историю коммитов в вашем репозитории Git.
Отображает список коммитов в обратном хронологическом порядке, начиная с последнего коммита, и показывает информацию о
каждом коммите, такую как идентификатор коммита, автор, дата и время коммита, и сообщение коммита.

- git remote -v отображает список удаленных репозиториев, связанных с вашим локальным репозиторием.
Показывает URL-адрес каждого удаленного репозитория и его имя. Например:
origin	git@github.com:SerZab/git-test.git (fetch)
origin	git@github.com:SerZab/git-test.git (push)
